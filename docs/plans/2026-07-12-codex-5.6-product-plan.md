# Codex 5.6 product plan

Date: 2026-07-12

## Product stance

Cogpit should be a provider-neutral local agent control center, not a Claude
history reader with Codex bolted on. Provider is visible context and a source
of capabilities; projects, activity, attention, and outcomes are the primary
navigation.

The core product promise is:

> See every agent, understand what needs attention, and safely steer work from
> one place without losing the provider's native capabilities.

## Sources of truth

Use sources in this order:

1. The model and capability catalog reported by the installed Codex app-server.
2. The app-server protocol generated by the installed Codex CLI.
3. Current official OpenAI product documentation and changelog.
4. Static fallbacks in Cogpit, used only when the live catalog is unavailable.

This ordering matters during staged rollouts. Documentation can lag the
capabilities already advertised by a user's installed CLI or account.

## GPT-5.6 model experience

The live catalog drives this matrix. The values below reflect Codex CLI 0.144.1
on 2026-07-12 and remain fallbacks, not permanent assumptions.

| Model | Product role | Reasoning | Inputs | Fast | Personality |
| --- | --- | --- | --- | --- | --- |
| GPT-5.6 Sol | Flagship for ambitious work | Light through Max, plus Ultra | Text and image | When advertised | No |
| GPT-5.6 Terra | Balanced everyday model | Light through Max, plus Ultra | Text and image | When advertised | No |
| GPT-5.6 Luna | Fastest and lowest cost | Light through Max | Text and image | When advertised | No |

Product rules:

- Default to the model's reported recommended effort, currently Medium for the
  GPT-5.6 family.
- Explain Ultra as proactive multi-agent delegation, not merely “more
  thinking.” Never show Ultra for a model or account that does not advertise it.
- Show Standard/Fast only when the selected model advertises a speed tier.
- Accept pasted or dropped images only when the selected model advertises image
  input.
- Preserve model descriptions, availability messages, modalities, service
  tiers, default state, and personality support from the live catalog.
- Use published prices for Sol ($5/$30), Terra ($2.50/$15), and Luna ($1/$6)
  per million input/output tokens. GPT-5.6 cache writes are 1.25x input and
  cache reads receive the 90% discount. Show no dollar estimate for an unknown
  GPT model rather than applying an unrelated fallback price.

GPT-5.6 also introduces Programmatic Tool Calling, explicit prompt-cache
controls, persisted reasoning, Pro mode, original-dimension image detail, and
beta multi-agent orchestration in the Responses API. In Cogpit:

- Programmatic/nested tool activity and persisted reasoning should render when
  app-server exposes native items; they are not artificial composer toggles.
- Prompt-cache breakpoints belong in advanced API/provider configuration once
  Codex exposes a supported thread-level contract.
- Pro is shown only if the Codex model catalog advertises it; ChatGPT-only
  availability is not enough.
- Original image detail becomes an attachment-level choice after the live
  Codex protocol advertises support. Auto detail remains the safe default.
- Multi-agent is represented as real agent threads and activity, not flattened
  text.

## Information architecture

```text
Activity
├── Needs attention
├── Running
├── Completed
└── History

Projects
└── Project (one item per working directory)
    ├── Threads by provider
    ├── Worktrees and workspace roots
    └── Agent trees

Thread
├── Goal and budget
├── Current plan
├── Activity timeline
├── Composer
└── Inspector
    ├── Changes
    ├── Agents
    ├── Processes
    ├── Context and usage
    └── Details

Library
├── Models
├── Skills
├── Apps
├── MCP
├── Hooks
├── Plugins
├── Agent configuration
└── Permission profiles
```

Composer controls should be organized by user intent:

- Mode: Default, Plan, Review.
- Intelligence: Auto by default; model, effort, and Fast in advanced controls.
- Access: Read only, Workspace, Full access, or a named permission profile.
- Context: files, images, skills, apps, and additional workspace roots.

When a Codex turn is active, the composer must explicitly distinguish **Steer
active turn** from **Queue next turn**. Interrupt remains immediately visible.
Full access is a two-step choice and stays visible in the thread chrome.

## Capability coverage

### Foundation — implemented in the GPT-5.6 catch-up slice

- Live model discovery with capability metadata and resilient static fallbacks.
- Model-specific effort defaults and Ultra gating.
- Capability-gated Fast and image input.
- Provider-specific access language with Workspace as the safe default and a
  confirmation step for Full access.
- Accurate GPT-5.6 pricing and native cached-input token accounting.
- No fabricated USD estimate for unknown GPT models.
- Persistent app-server transport with request correlation, restart, timeouts,
  notifications, active-turn tracking, explicit replies for unsupported server
  requests, and command/file approval handling.
- Native thread, turn, steering, interruption, goal, runtime, account usage,
  and permission endpoints, with legacy CLI fallback where appropriate.
- Goal strip with create/edit, pause/resume/complete/clear, optional token
  budget, tokens used, and elapsed time. Goal and plan remain separate.
- Per-request approval decisions, server-side validation, and nested-subagent
  approval aggregation into the open parent thread.
- Current Codex multi-agent JSONL parsing, including lifecycle events,
  namespaced tools, routed agent messages, and deduplication.
- Codex-only onboarding through `CODEX_HOME` or `~/.codex`.
- Truthful “Live & recent” session counts and local session search.

### Native core — next

- Make app-server the only live Codex control plane after a compatibility
  window; retain JSONL as a history and offline-read format.
- Add a persistent Needs Attention inbox for command, file, network,
  permission-escalation, MCP elicitation, and user-input requests.
- Render native app-server thread items rather than flattening everything into
  Claude-shaped Bash, TodoWrite, and assistant text records.
- Thread list/read/search/start/resume/fork/archive/unarchive/delete/name.
- Clear separation between **Fork conversation** and **Restore workspace**.
- Native plan updates, compaction, rollback, and output-schema turns.
- First-class review targets: working tree, base branch, commit, and GitHub pull
  request.

### Differentiated Cogpit experience

- One project per working directory with provider chips instead of duplicate
  Claude and Codex project cards.
- Activity-first dashboard with Running, Needs Attention, Completed, and
  History filters.
- Thread inspector for changes, subagents, processes, context, and usage.
- Live multi-agent tree with path, nickname, role, state, current action,
  elapsed time, token usage, inspect, steer, interrupt, and close.
- Provider-aware authentication, plan, credits, rate limits, and token-usage
  history from app-server.
- Unified Library backed by `skills/list`, `app/list`, MCP status/tools, hooks,
  plugins, configuration, AGENTS.md, rules, and permission profiles.
- Composer mentions: `$` for skills and `@` for files/apps, following the
  provider's reported support.
- Named permission profiles and visible guardian/auto-review state.
- Multi-root and multi-repository projects, worktree actions, integrated
  terminal/background processes, and environment selection.
- Browser, web search, computer use, image generation, appshots, and browser
  annotations rendered as native timeline items with previews and evidence.
- Inline markdown/code editing and review annotations without obscuring the
  underlying file diff.

### Advanced surfaces

- Remote control status, pairing, client management, and mobile approvals.
- Realtime voice and transcript controls.
- Scheduled automations, cloud tasks, and notifications.
- Memory and Chronicle controls when their product contracts are stable.
- External-agent import and migration diagnostics.
- Plugin sharing and marketplace flows. Label custom app-server plugin
  management Preview until the protocol reaches stable maturity.

## Architecture decisions

1. **App-server owns live Codex work.** It is the official deep-integration
   interface for auth, history, approvals, and streaming. `codex exec --json`
   remains a temporary fallback for older CLIs.
2. **Store native and normalized event data.** Native method/item type is never
   discarded. Normalized presentation metadata enables a coherent shared UI.
3. **Gate by capability, not provider-name conditionals.** Provider checks are
   acceptable only at transport boundaries.
4. **Safe defaults are non-negotiable.** Workspace access is default. Full
   machine access requires an explicit, visible confirmation.
5. **Usage and dollar cost are different.** ChatGPT credits/rate limits come
   from account APIs. USD values are estimates only when published model prices
   are known.
6. **One project identity per working directory.** Threads retain provider and
   source metadata beneath the project.
7. **Feature maturity is visible.** Stable, beta, preview, and unavailable
   states are represented in the UI; unsupported controls are not silently
   ignored.

## Release gates

A Codex capability is complete only when:

- It is discoverable in the intended desktop and mobile flow.
- Unsupported models/accounts cannot select it.
- Permission and failure states have an actionable UI.
- Live operation and restored history render consistently.
- Interrupt/reconnect/restart paths are tested.
- Claude behavior is covered by regression tests.
- Token/credit/cost labels state what is measured.
- The feature's maturity and fallback behavior are documented.

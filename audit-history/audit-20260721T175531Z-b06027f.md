# Audit: Lazy Session Loading & Session Config Persistence (2026-07-21)

**Commit ID:** b06027f  
**Scope:** Lazy session loading unification (perf) + session config persistence & live permission application (feat)

## Implementation Summary

### Lazy Session Loading Unification
- New `src/lib/sessionLoader.ts` module consolidates tail-load pattern with cache and worker parsing
- All session-open paths (sidebar browser, URL deep-link, reload, branch, duplicate) now use unified `loadSessionTailCached()` / `loadSessionTailFresh()` instead of whole-file main-thread reads
- Server-side `?tail=30` + `?before=&count=` endpoint support (already exists) now serves all clients equally
- Branch-from-here sends `turnUuid` to prevent index-misalignment on tail-loaded sessions; server cuts at correct turn boundary
- In-session search background-hydrates full transcript when activated
- Non-virtual timeline gained "Load older turns" button for chunked scroll-up
- Fixes: `useSessionBrowser`, `useUrlSync`, `useAppHandlers` (reload/branch/duplicate), related tests
- No new API routes; existing `GET /api/sessions/:dir/:file?tail=/before=` endpoints carry all traffic

### Session Config Persistence & Live Permission Application
- NEW server route: `GET/PUT /api/session-config/:key` (server/routes/session-config.ts, registered in api-routes.ts)
- Stores per-session UI configuration (model, effort, permission mode, MCP selection) server-side in gitignored `session-config/` directory
- Client: `src/lib/sessionConfig.ts` (debounced merge-PUT) + `useSessionConfigSync` hook keep composer settings synced across devices/clients
- MCP selection synced via `useMcpServers` + `useSessionConfigSync`
- sdk-session.ts: switching to `bypassPermissions`/`acceptEdits` mid-turn auto-resolves matching pending permission requests
- **Breaking change:** `DEFAULT_PERMISSIONS.mode` changed from `"default"` to `"bypassPermissions"` (full access now default)
- **Breaking change:** full-access confirmation dialog removed (product decision; no UI barrier to full access)

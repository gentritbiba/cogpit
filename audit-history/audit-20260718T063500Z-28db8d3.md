**2. Leak Monitor (Pre-Existing Work, Committed Alongside):**
- **Server-side system-wide agent process scan** (`server/lib/systemProcesses.ts`):
  - Parses `ps` output classifying processes by kind (claude, headless-browser, browser-daemon, cogpit, script, other).
  - Detects suspected leaks: orphaned Claude sessions (parent dead, reparented to launchd), hot headless browsers (≥10% CPU, ≥30 min age), orphaned scripts.
  - Returns top 20 processes sorted by leak risk.
- **Two-sweep leak reaper** (`server/lib/leakReaper.ts`):
  - Polls every 10 minutes (first sweep after 1 minute).
  - Requires two consecutive sweeps to see same orphan before killing (prevents transient/recycled-pid kills).
  - Only auto-reaps unambiguously safe targets: orphaned Claude sessions ≥30 min old.
  - Hot browsers and orphaned scripts surfaced for manual kill in UI.
  - Auto-sends desktop notification on reap.
  - Maintains reap log (last 20 events).
- **Desktop notifications** (`server/lib/desktopNotify.ts`):
  - Extracted from `server/routes/notify.ts` for reuse by leak reaper and other subsystems.
  - Electron Notification API with fallback to osascript on non-Electron.
- **API routes** (registered in `server/routes/performance.ts`):
  - `GET /api/system-processes` → Returns `SystemProcessesSnapshot` + `recentlyReaped` array.
  - `POST /api/system-processes/kill` → Kills PIDs after whitelist-check against current suspects (prevents targeting Cogpit/live sessions).
  - Enhanced `GET /api/performance` → Now includes system snapshot (optional, best-effort).
- **Frontend leak monitor** (`src/hooks/useLeakMonitor.ts`):
  - Polls `GET /api/system-processes` every 60s (visibility-gated, only when tab is active).
  - Returns suspected leaks with CPU/memory/age data for display.
  - `killLeaks()` function calls `POST /api/system-processes/kill` endpoint.
- **LeakIndicator component** (header badge):
  - Muted with count=0 when clean.
  - Red with count when leaks present.
  - Click to kill all suspected leaks.
  - Tooltip shows top 6 leaks (CPU%, age, label).
  - Pulsing flame icon while killing.
- **PowerMonitor enhancement**:
  - Displays system-wide process list in dedicated panel view.
- **Type additions** (`src/lib/performanceTypes.ts`):
  - `SystemProcessKind`, `SystemProcessMetric`, `SystemProcessesSnapshot`, `ReapedEvent`, `SystemProcessesResponse`.
  - Enhanced `ServerPerformanceSnapshot` with optional `system` field.

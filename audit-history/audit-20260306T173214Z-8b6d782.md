## Changes Detected

### 1. Cogpit-Search Route Refactored to Shell Out CLI

**File:** `server/routes/cogpit-search.ts` (MAJOR REFACTOR)

**What Changed:**
- **Old approach:** Direct import of cogpit-memory module functions
- **New approach:** Spawn child process running CLI via `execFile()`, parse JSON stdout
- **Dual-path logic:** Development vs production executable detection (lines 17-31)
  - Dev: `bun packages/cogpit-memory/src/cli.ts` (fast reload, uses source TypeScript)
  - Production: `node packages/cogpit-memory/dist/cli.js` (compiled, from app.asar.unpacked)

**New Functions:**
```typescript
function getCliConfig(): { executable: string; script: string }
  - Checks for dev TypeScript source first
  - Falls back to production compiled JS
  - Uses app.asar/app.asar.unpacked path detection

function runCli(args: string[]): Promise<string>
  - Spawns execFile with timeout: 30_000ms
  - Sets ELECTRON_RUN_AS_NODE=1 environment variable
  - Rejects on stderr, resolves on stdout
```

**Impact:**
- No API contract changes (route signature identical)
- Internal implementation change only
- Response shape remains `{ results: [...] }`
- Error handling preserved (stderr becomes error message)

**Why This Matters:**
- Enables packaging cogpit-memory as separate distribution (not bundled into main executable)
- Faster app startup (lazy-load heavy indexing library)
- Allows cogpit-memory to be updated independently

**Documentation Impact:**
- ARCHITECTURE.md section "Cogpit Session Search" (line 119-131) mentions "local `packages/cogpit-memory` indexing package"
- New detail: Now shells out to CLI instead of direct import
- **Update needed:** Add note about CLI execution and asarUnpack configuration

**What This Does:**
- Extracts cogpit-memory/dist/ files from the ASAR archive into `app.asar.unpacked/`
- Allows cogpit-search route to spawn CLI executable from unpacked location
- Keeps compiled CLI files readable by node process

**Why This Matters:**

- Unpacking cogpit-memory dist enables CLI spawning without extraction overhead

### 3. New API Route: /api/file-content

**File:** `server/routes/file-content.ts` (NEW)

**What It Does:**
- GET endpoint for reading text file contents
- Query parameter: `path` (required, must be absolute)
- Returns plain text response with Content-Type: text/plain
- Size limit: 2MB
- Validation: must be file (not directory), must exist

**Handler Details:**
```typescript
registerFileContentRoutes(use: UseFn)
  GET /api/file-content?path=/absolute/path/to/file
  - Returns: 200 with file content (plain text)
  - Returns: 400 if path missing or not absolute
  - Returns: 404 if file not found or is directory
  - Returns: 413 if file > 2MB
  - Returns: 500 if read fails
  - Sets: Cache-Control: no-cache
```

**Use Cases:**
- LiveSessions or other UI components displaying file content inline
- Image content inspection (when served as data URLs)
- Log file viewing

**Registration Status:**

- ✓ `electron/server.ts` line 132: `registerFileContentRoutes(use)`
- ✓ `server/api-plugin.ts` line 26: `import { registerFileContentRoutes }`
- ✓ `server/api-plugin.ts` line 107: `registerFileContentRoutes(use)`
